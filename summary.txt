The LangChain Team has released the LangSmith CLI and a first set of skills to empower AI coding agents within the LangSmith ecosystem. This initiative aims to equip agents with expertise in adding tracing, understanding execution, building test sets, and evaluating performance. Notably, this improved Claude Code's performance on these tasks significantly, from 17% to 92%. The LangSmith CLI is agent-native, offering coding agents and developers the building blocks for operations like fetching traces, curating datasets, and running experiments. It can be installed using `curl -sSL https://raw.githubusercontent.com/langchain-ai/langsmith-cli/main/scripts/install.sh | sh`.

Skills are curated instructions, scripts, and resources that enhance coding agent performance in specialized domains. They are dynamically loaded through progressive disclosure, meaning agents retrieve skills only when relevant, preventing performance degradation from too many tools. Skills are portable, shareable markdown files and scripts. The initial `langsmith-skills` repo includes 3 skills: `trace` for adding tracing to existing code, `dataset` for building example datasets, and `evaluator` for evaluating agents over those datasets. These skills cover the three core areas of LangSmith AI engineering.

Utilizing these, agents can create a virtuous cycle: add tracing logic, generate traces for debugging, use traces to build systematic testing datasets, create evaluators, and iterate based on evaluations. Skills can be installed locally with `npx skills add langchain-ai/langsmith-skills --skill '*' --yes` or globally with `npx skills add langchain-ai/langsmith-skills --skill '*' --yes --global`. To link skills to a specific agent like Claude Code, use `npx skills add langchain-ai/langsmith-skills --agent claude-code --skill '*' --yes --global`. The team plans to continuously add skills content and is also releasing skills for LangChain's open-source libraries.
